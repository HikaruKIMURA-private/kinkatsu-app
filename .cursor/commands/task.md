# Task Generator Command

## 概要

このコマンドは、プロジェクトの要件に基づいて実装タスクを生成するためのものです。

## 使用方法

ユーザーから指示された「`${ARGUMENTS}`」 と `.cursor/rules` を参照して、プロジェクトルートに `task.md` を生成します。

## 生成ルール

### 基本方針

- **出力先**: `task.md` はプロジェクトルートに生成する
- **機能単位分割**: Phase/Task は機能単位で分割し、技術的な層（DB/API/Frontend）で分けない
- **実装とセット**: 型定義のみではレビュー可能な単位にならないため、実装とセットでレビューできる単位に分割する
- **段階的実装**: 大きな単位（Phase）と小さな単位（Task）に分けて管理する

### 構成例

```
Phase 1: チケット一覧機能
├── Task 1.1: データベース設計とマイグレーション
├── Task 1.2: 型定義とAPI実装
├── Task 1.3: フロントエンド実装
└── Task 1.4: 統合テストとレビュー

Phase 2: チケット詳細・編集機能
├── Task 2.1: API実装（詳細取得・更新）
├── Task 2.2: フロントエンド実装
└── Task 2.3: 統合テストとレビュー

Phase 3: コメント・履歴機能
├── Task 3.1: API実装（コメント・履歴）
├── Task 3.2: フロントエンド実装
└── Task 3.3: 統合テストとレビュー

```

### 品質保証

- **テスト**: 各 Task ごとにユニットテストを実装し、パスすることを確認する
- **レビュー**: Phase ごとに人間によるコードレビューを実施し、承認を得てから次の Phase に進む

## 注意事項

- **機能単位分割**: Phase/Task は機能単位で分割し、技術的な層（DB/API/Frontend）で分けない
- **実装とセット**: 型定義のみではレビュー可能な単位にならないため、実装とセットでレビューできる単位に分割する
- **独立実装**: 各 Task は独立して実装可能な単位に分割する
- **依存関係**: 依存関係を明確にし、実装順序を考慮する
- **テスト可能**: テスト可能な形でタスクを定義する
